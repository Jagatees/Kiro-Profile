# Security Policy

## Supported versions

Security fixes are applied to the latest published version.

## Reporting a vulnerability

Do not open a public issue for secrets exposure, unsafe leaderboard publishing, path disclosure, or webview injection. Use GitHub's private security advisory flow for this repository and include reproduction steps, affected version, and impact. Maintainers will acknowledge a complete report within seven days.

Kiro Stat must never publish prompts, responses, emails, ARNs, API keys, file content, workspace names, or local paths.
